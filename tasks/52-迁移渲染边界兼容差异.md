# 52.迁移渲染边界兼容差异

## 目标

- 迁移 Vue 3 中与渲染边界相关的剩余兼容差异：`$attrs` 包含 `class` / `style`、`TransitionGroup` 默认无根元素、`Transition` 作为组件根不再响应外部切换、VNode 生命周期事件前缀变化。
- 保持样式落点、动画触发、列表过渡和生命周期监听语义不变。
- 默认保持现有业务行为和组件外部 API。

## 最小改动原则

- 能不改就不改。
- 不能证明某处代码会阻塞 Vue3 迁移时，不修改，只记录。
- 不做命名、格式、目录、架构、业务逻辑优化。
- 不因为代码看起来旧、乱、重复就顺手重构。
- 不把多个边界问题合并为组件重写；只处理明确命中的兼容差异。

## 允许修改范围

- 只修改本任务搜索命令命中的业务源码、配置或依赖声明。
- 只修改与本任务目标直接相关的最小代码片段。
- 如果需要修改公共组件 API，必须先记录调用方影响，并停止等待人工确认。
- 如果命中构建、入口或依赖配置，只修改能证明是 Vue3 迁移阻塞的配置项。

## 禁止修改范围

- 不引入 `@vue/compat`。
- 不 patch `node_modules`。
- 不顺手升级无关依赖。
- 不迁移到 Vite、Pinia 或 Composition API。
- 不修改无关格式。

## 搜索命令

原搜索线索：全局搜索 `inheritAttrs`、`v-bind="$attrs"`、`transition-group`、根 `<transition>`、`@hook:` 生命周期事件。

```bash
rg -n 'inheritAttrs|\$attrs|<transition-group|<transition\b|@hook:|v-on:hook:' src
```

## 命中分类

- 必改：`inheritAttrs: false` 且 `v-bind="$attrs"` 落到内部元素，样式原本依赖 Vue 2 自动落到根元素。
- 必改：`<transition-group>` 没有 `tag`，且 CSS、选择器或布局依赖 Vue 2 默认 `<span>` 根元素。
- 必改：组件模板根节点是 `<transition>`，父组件通过 `v-if` / `v-show` 切换该组件并依赖内部过渡。
- 必改：模板中使用 `@hook:*` 或 `v-on:hook:*` 监听 VNode 生命周期事件。
- 可不改：`$attrs` 的 class/style 本来就应该落到当前绑定元素。
- 可不改：`transition-group` 已显式声明 `tag`。
- 阻塞：无法判断 class/style 应落到根元素、表单控件还是子组件。
- 阻塞：过渡组件涉及复杂缓存、路由切换或第三方动画库。

## 修改规则

- `$attrs`：明确拆分或移动 `class` / `style` 落点，保证原样式仍作用在预期元素。
- `transition-group`：如果依赖默认根元素，补 `tag="span"`；如果原布局应无根元素，保持并记录。
- 根 `transition`：将外部切换改为传入 prop，并在组件内部切换 transition 的子元素；无法确认外部 API 时停止。
- VNode 生命周期事件：`@hook:updated` 改为 `@vue:updated`，`hook:beforeDestroy` / `hook:destroyed` 同步改为 `vue:beforeUnmount` / `vue:unmounted`。
- 对“必改”命中做最小兼容性修改。
- 对“可不改”命中保持原样，并在完成报告中说明。
- 对“阻塞”命中停止修改，记录文件、行号、原因和建议人工决策点。

- 不通过 webpack alias、降级依赖、复制源码等方式绕过问题。
- 只在任务明确允许时修改项目内自研适配层。

## 人工验收点

- 包装组件的 `class`、`style`、`id`、`disabled` 等属性落点正确。
- 列表过渡布局、动画类和选择器仍匹配。
- 弹窗、抽屉、路由页等以 `transition` 为根的组件切换动画仍触发。
- VNode 生命周期监听只触发一次，且时机与原业务预期一致。

## 停止条件

- 需要改变组件外部 API。
- 需要同时修改大量调用方才能保持行为。
- 命中第三方库或私有插件且兼容策略不明确。
- 无法判断某处代码是业务字段、普通事件还是 Vue2 迁移目标。
- 修改后出现与本任务无关的大面积 diff。
