# 45.迁移异步组件定义

## 目标

- 将 Vue 2 异步组件定义迁移为 Vue 3 的 `defineAsyncComponent`。
- 区分普通异步组件和 vue-router 懒加载路由组件，避免错误包装路由 component。
- 默认保持现有 loading、error、delay、timeout 行为。

## 最小改动原则

- 能不改就不改。
- 不能证明某处代码会阻塞 Vue3 迁移时，不修改，只记录。
- 不做命名、格式、目录、架构、业务逻辑优化。
- 不因为代码看起来旧、乱、重复就顺手重构。
- 路由懒加载仍保持 `() => import(...)`，不要套 `defineAsyncComponent`。

## 允许修改范围

- 只修改本任务搜索命令命中的业务源码、配置或依赖声明。
- 只修改与本任务目标直接相关的最小代码片段。
- 如果需要修改公共组件 API，必须先记录调用方影响，并停止等待人工确认。
- 如果命中构建、入口或依赖配置，只修改能证明是 Vue3 迁移阻塞的配置项。

## 禁止修改范围

- 不引入 `@vue/compat`。
- 不 patch `node_modules`。
- 不顺手升级无关依赖。
- 不迁移到 Vite、Pinia 或 Composition API。
- 不修改无关格式。

## 搜索命令

原搜索线索：全局搜索返回 `import()` 的组件定义、带 `component` / `loading` / `error` / `delay` / `timeout` 的异步组件对象。

```bash
rg -n '=>\s*import\(|component\s*:\s*\(\)\s*=>\s*import\(|component\s*:\s*function|loading\s*:|error\s*:|delay\s*:|timeout\s*:' src
```

## 命中分类

- 必改：局部或全局组件注册中使用 `() => import('./Comp.vue')` 作为组件定义。
- 必改：异步组件对象使用 `component: () => import(...)`、`loading`、`error`、`delay`、`timeout`。
- 必改：异步组件 loader 形如 `(resolve, reject) => { ... }`，需要改成返回 Promise。
- 可不改：vue-router 路由表中的 `component: () => import(...)`。
- 可不改：普通业务动态 import，不作为 Vue 组件使用。
- 阻塞：无法判断某个 import 是路由组件、普通模块还是异步组件。
- 阻塞：异步组件 loading/error 组件来自第三方封装且行为不清。

## 修改规则

- 从 `vue` 引入 `defineAsyncComponent`。
- 简单异步组件改为 `defineAsyncComponent(() => import('./Comp.vue'))`。
- 高级异步组件对象中 `component` 改为 `loader`，`loading` 改为 `loadingComponent`，`error` 改为 `errorComponent`。
- loader 不再接收 `resolve` / `reject` 参数，必须返回 Promise。
- 路由懒加载组件不套 `defineAsyncComponent`，只在完成报告中说明保留原因。
- 对“必改”命中做最小兼容性修改。
- 对“可不改”命中保持原样，并在完成报告中说明。
- 对“阻塞”命中停止修改，记录文件、行号、原因和建议人工决策点。

## 第三方依赖处理

- 如果命中来自 `node_modules`、vendor 包、UI 组件库包装层或私有插件，先记录为阻塞。
- 不通过 webpack alias、降级依赖、复制源码等方式绕过问题。
- 只在任务明确允许时修改项目内自研适配层。

## 验证命令

```bash
rg -n 'component\s*:\s*\(\)\s*=>\s*import\(|component\s*:\s*function|loading\s*:|error\s*:|defineAsyncComponent' src
git diff --check
git diff --stat
```

如果搜索范围与项目实际目录不一致，把命令中的路径替换为实际源码、配置或依赖文件范围。若项目有明确脚本，再运行项目实际脚本，例如：

```bash
npm run lint
npm run test
npm run build
```

## 人工验收点

- 异步弹窗、异步页面片段、低频组件首次打开能正常加载。
- loading、error、delay、timeout 行为与原业务预期一致。
- 路由懒加载仍能正常分包和跳转。

## 停止条件

- 需要改变组件外部 API。
- 需要同时修改大量调用方才能保持行为。
- 命中第三方库或私有插件且兼容策略不明确。
- 无法判断某处代码是业务字段、普通事件还是 Vue2 迁移目标。
- 修改后出现与本任务无关的大面积 diff。

- 剩余命中数量和保留原因。
- 运行过的验证命令及结果。
- 阻塞项和需要人工验收的页面或交互。
