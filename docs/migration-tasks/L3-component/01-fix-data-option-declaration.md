# Task L3-01: 修复 data 选项声明方式

## 基本信息

- **任务ID**: L3-01
- **层级**: L3 - 组件定义层
- **优先级**: P0
- **影响范围**: 组件
- **复杂度**: 中
- **预计工时**: 4-8小时

## 任务描述

Vue 3 中 `data` 选项必须是一个返回对象的函数，不再接受纯对象声明。查找所有使用对象声明的 `data: {}` 并改为函数声明。

## 迁移指南

- [Data Option](https://v3-migration.vuejs.org/breaking-changes/data-option.html)

## 检查清单

- [ ] 查找所有使用对象声明的 `data: {}`
- [ ] 改为函数声明 `data() { return {} }`
- [ ] 检查根实例的数据声明

## 代码对比

### Vue 2

```js
export default {
  data: {
    count: 0
  }
}
```

### Vue 3

```js
export default {
  data() {
    return {
      count: 0
    }
  }
}
```

## 常见问题

1. **根实例的特殊处理**：在 Vue 2 中根实例可以使用对象声明，但 Vue 3 中统一要求函数声明
2. **共享数据问题**：如果之前依赖对象声明来共享数据，需要改为使用状态管理或其他方式

## 注意事项

1. 确保所有组件的 `data` 都是函数
2. 检查是否有代码依赖 `data` 的对象引用

## 验收标准

- [ ] 所有组件的 `data` 选项为函数形式
- [ ] 应用启动无警告
- [ ] 组件数据正常响应
