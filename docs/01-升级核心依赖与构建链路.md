# 01. 升级核心依赖与构建链路

## 当前任务

先把项目升级到能编译 Vue 3 的基础状态，后面的 breaking changes 才有机会逐条修改。

## 怎么做

1. 保持现有 Vue CLI 路线，只升级 `@vue/cli-service` 和现有 webpack 链路，不切换构建工具。
2. 修改 `package.json`，至少同时处理这几项：`vue` 升到 3.x，删除 `vue-template-compiler`，新增 `@vue/compiler-sfc`，升级 `@vue/cli-service`。
3. 如果项目显式安装了 `vue-loader`，一起升级到支持 Vue 3 的版本。
4. 执行一次依赖安装，先把 `peer dependency` 冲突处理干净。
5. 检查 `vue.config.js` 和 webpack 配置，确认 `.vue` 文件编译已经走 `@vue/compiler-sfc`，不再依赖 Vue 2 编译链。

## 改完以后确认

- 依赖安装成功。
- 本地开发命令至少能进入编译阶段。
